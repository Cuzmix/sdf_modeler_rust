--- Iteration 1: PRD #6 — vec2f → vec4f pipeline (material blending infrastructure) ---
COMPLETED. Expanded scene_sdf return type from vec2f to vec4f across the entire pipeline.

Changes:
- src/shaders/operations.wgsl: All 4 CSG ops (op_union, op_smooth_union, op_subtract, op_intersect)
  now take/return vec4f(distance, mat_a, mat_b, blend_factor). Hard ops set mat_b=-1.0, blend=0.0.
  Smooth union stores both material IDs and blend factor h for future material interpolation.
- src/gpu/codegen.rs: emit_node_wgsl emits vec4f for all node types (Primitive, Operation, Sculpt,
  Transform, Modifier). Primitives: vec4f(dist, mat_id, -1.0, 0.0). Transforms/modifiers propagate
  all 4 components (.y, .z, .w). Empty/fallback: vec4f(1e10, -1.0, -1.0, 0.0).
  generate_scene_sdf signature: -> vec4f. Composite render shader scene_sdf also returns vec4f.
- Updated 6 codegen unit tests to assert on vec4f instead of vec2f.
- rendering.wgsl and pick.wgsl: NO changes needed — they access .x (distance) and .y (mat_id)
  which are unchanged in the vec4f layout. Zero visual regression.
