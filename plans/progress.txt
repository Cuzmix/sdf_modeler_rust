--- Iteration 1: PRD #6 — vec2f → vec4f pipeline (material blending infrastructure) ---
COMPLETED. Expanded scene_sdf return type from vec2f to vec4f across the entire pipeline.

Changes:
- src/shaders/operations.wgsl: All 4 CSG ops (op_union, op_smooth_union, op_subtract, op_intersect)
  now take/return vec4f(distance, mat_a, mat_b, blend_factor). Hard ops set mat_b=-1.0, blend=0.0.
  Smooth union stores both material IDs and blend factor h for future material interpolation.
- src/gpu/codegen.rs: emit_node_wgsl emits vec4f for all node types (Primitive, Operation, Sculpt,
  Transform, Modifier). Primitives: vec4f(dist, mat_id, -1.0, 0.0). Transforms/modifiers propagate
  all 4 components (.y, .z, .w). Empty/fallback: vec4f(1e10, -1.0, -1.0, 0.0).
  generate_scene_sdf signature: -> vec4f. Composite render shader scene_sdf also returns vec4f.
- Updated 6 codegen unit tests to assert on vec4f instead of vec2f.
- rendering.wgsl and pick.wgsl: NO changes needed — they access .x (distance) and .y (mat_id)
  which are unchanged in the vec4f layout. Zero visual regression.

--- Iteration 2: PRD #7 — Consume material blend info in fragment shader ---
COMPLETED. Material properties now interpolate smoothly at CSG boundaries.

Changes (src/shaders/rendering.wgsl only):
- ray_march() now returns vec4f(t, mat_a, mat_b, blend_factor) instead of vec2f(t, mat_id).
  Captures all 4 components from scene_sdf: distance, primary material, secondary material,
  and blend factor. Miss sentinel: vec4f(MAX_DIST+1, -1, -1, 0).
- Added 4 blending helper functions:
  - get_blended_color(mat_a, mat_b, blend_factor) -> vec3f
  - get_blended_roughness(mat_a, mat_b, blend_factor) -> f32
  - get_blended_metallic(mat_a, mat_b, blend_factor) -> f32
  - get_blended_fresnel(mat_a, mat_b, blend_factor) -> f32
  Each returns mat_a's property unchanged when mat_b < 0 or blend_factor <= 0.
  Otherwise mixes both materials' properties using WGSL mix().
- Full shading path: replaced direct nodes[mat_id] lookups for metallic, roughness,
  fresnel, and albedo with blended versions using all 3 material components.
- Solid shading path: replaced get_node_color(mat_id) with get_blended_color().
- Clay/Normals/Matcap/Heatmap modes: unchanged (don't use per-node materials).
- pick.wgsl: unchanged (pick only needs .x/.y, not material blending).

--- Iteration 5: PRD #10 — Add WGSL noise functions for procedural effects ---
COMPLETED. Added noise shader functions for use by future domain warp modifier.

Changes:
- Created src/shaders/noise.wgsl with 4 pure GPU functions:
  - hash33(vec3f) -> vec3f: hash-based pseudo-random using sin/fract with large primes
  - quintic(vec3f) -> vec3f: C2-continuous interpolation curve (6t^5 - 15t^4 + 10t^3)
  - noise3d(vec3f) -> f32: 3D gradient noise with trilinear interpolation of hashed
    gradients at 8 cube corners, smoothed with quintic fade
  - fbm_noise(vec3f, f32, f32, i32) -> vec3f: fractal Brownian motion returning 3D
    displacement. Lacunarity=2.0, persistence=0.5, max 8 octaves. Three axes use
    different hash offsets to decorrelate displacement components.
- src/gpu/shader_templates.rs: Added NOISE constant (include_str noise.wgsl). Included
  in both render_prelude() and compute_prelude() arrays, positioned after MODIFIERS
  and before OPERATIONS. This makes noise functions available in render, pick, and
  composite shaders.

--- Iteration 6: PRD #11 — Noise domain warp modifier ---
COMPLETED. Added Noise modifier for organic shape deformation via FBM domain warping.

Changes:
- src/graph/scene.rs: Added ModifierKind::Noise variant. is_point_modifier=true
  (domain warp — modifies point before child eval). Default value Vec3(2.0, 0.1, 3.0)
  encoding (frequency, amplitude, octaves). gpu_type_id=41.0. Badge "[Nse]".
  Added to ModifierKind::ALL.
- src/gpu/codegen.rs: For Noise point modifier, emits:
  'let {var} = {parent_var} + fbm_noise({parent_var}, freq, amp, i32(octaves))'
  where freq/amp/octaves are read from nodes[idx].position.x/.y/.z.
  Added codegen test: noise_modifier_emits_fbm_noise.
- src/graph/voxel.rs: Implemented CPU-side noise functions matching WGSL noise.wgsl:
  - hash33(Vec3) -> Vec3: same sin/fract hash with identical magic constants
  - quintic(Vec3) -> Vec3: C2-continuous interpolation curve
  - noise3d(Vec3) -> f32: 3D gradient noise with trilinear interpolation
  - fbm_noise(Vec3, f32, f32, i32) -> Vec3: FBM with decorrelated 3-axis displacement
  Noise modifier applies fbm displacement to sampling point before evaluating child SDF.
  Added 2 tests: displacement with non-zero amplitude, identity with zero amplitude.
- src/ui/properties.rs: Added 3 controls for Noise modifier — Frequency (0.1–20.0),
  Amplitude (0.0–2.0), Octaves (1–8 integer slider). Added Noise to "Deform" category

  Added to node graph toolbar "Deform" section.
