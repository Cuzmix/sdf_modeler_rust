Changes:
- src/shaders/operations.wgsl: All 4 CSG ops (op_union, op_smooth_union, op_subtract, op_intersect)

--- Iteration 5: PRD #10 — Add WGSL noise functions for procedural effects ---
COMPLETED. Added noise shader functions for use by future domain warp modifier.

Changes:
- Created src/shaders/noise.wgsl with 4 pure GPU functions:
  - hash33(vec3f) -> vec3f: hash-based pseudo-random using sin/fract with large primes
  - quintic(vec3f) -> vec3f: C2-continuous interpolation curve (6t^5 - 15t^4 + 10t^3)
  - noise3d(vec3f) -> f32: 3D gradient noise with trilinear interpolation of hashed
    gradients at 8 cube corners, smoothed with quintic fade
  - fbm_noise(vec3f, f32, f32, i32) -> vec3f: fractal Brownian motion returning 3D
    displacement. Lacunarity=2.0, persistence=0.5, max 8 octaves. Three axes use
    different hash offsets to decorrelate displacement components.
- src/gpu/shader_templates.rs: Added NOISE constant (include_str noise.wgsl). Included
  in both render_prelude() and compute_prelude() arrays, positioned after MODIFIERS
  and before OPERATIONS. This makes noise functions available in render, pick, and
  composite shaders.

--- Iteration 10: PRD #19 — Cook-Torrance GGX BRDF upgrade ---
COMPLETED. Replaced Blinn-Phong specular with physically-based Cook-Torrance microfacet BRDF.

Changes:
- src/shaders/rendering.wgsl: Added 3 Cook-Torrance BRDF functions (Filament reference):
  - D_GGX(NoH, roughness): GGX normal distribution. Formula: a^2 / (PI * (NoH^2*(a^2-1)+1)^2)
    where a = roughness^2. Controls highlight shape — tight for smooth, wide for rough.
  - G_SmithGGX(NoV, NoL, roughness): Height-correlated Smith geometric attenuation.
    Correlated form cancels the 4*NoV*NoL denominator: 0.5 / (NoL*sqrt(...) + NoV*sqrt(...)).
  - F_Schlick_vec3(VoH, f0): Schlick Fresnel returning vec3f for metallic F0 = albedo.
    Dielectric F0 = mat_fresnel, metallic F0 = albedo color. Mixed via metallic parameter.
  Added PI constant for diffuse normalization.
- Full shading path rewritten:
  - Diffuse: Lambertian (albedo / PI * (1 - metallic)). Metals have zero diffuse.
  - Specular: D * G * F (Cook-Torrance). No explicit / (4*NoV*NoL) — cancelled in G form.
  - Both key light AND fill light now use full Cook-Torrance BRDF (previously fill was
    diffuse-only). KEY_SPEC_INTENSITY preserved as artistic multiplier on specular terms.
  - Roughness clamped to [0.045, 1.0] minimum to avoid division artifacts at perfectly smooth.
  - F0 computed as mix(vec3f(mat_fresnel), albedo, mat_metallic) — industry standard.
  - Final accumulation: (diffuse + specular * spec_intensity) * NoL * light_intensity * shadow.
- src/gpu/shader_templates.rs: Removed KEY_SPEC_POWER placeholder replacement (Blinn-Phong
  concept, no longer needed — roughness drives highlight shape directly). Updated env_refl_line
  to use F_Schlick_vec3 and f0 instead of old mat_fresnel/spec_tint variables.
- Settings field key_spec_power retained for backwards compatibility (serialization), but
  no longer consumed by the shader. All other render settings preserved unchanged.
- Clay/Solid/Normals/Matcap/Heatmap modes: unchanged.

--- Iteration 12: PRD #29 — Undo snapshots for property edits on mouse release ---
COMPLETED. Verified that the existing History system already handles property edit
undo coalescing correctly. Added 4 comprehensive tests to prove the behavior.

Analysis: The PRD described this as a missing feature, but the frame-based History
system (begin_frame/end_frame with was_dragging tracking) already implements the
industry-standard approach. Here's how it works:
- begin_frame() captures scene state at frame start (pending snapshot)
- Property panels and gizmo modify scene directly via &mut Scene during the frame
- end_frame() receives is_anything_dragged from ctx.dragged_id().is_some()
- During drag: pending snapshot is preserved (no commit)
- When drag ends: drag_just_ended=true triggers commit of the pre-drag snapshot
- Result: entire drag coalesced into one undo entry, Ctrl+Z reverts full drag

This covers: property slider drags, gizmo transforms, color picker drags,
and instant click edits (which get immediate single-entry commits).

Changes:
- src/graph/history.rs: Added 4 unit tests + NodeData import:
  - property_slider_drag_coalesced_to_single_undo: simulates 3-frame slider drag,
    verifies exactly 1 undo entry, verifies undo restores original position
  - gizmo_drag_coalesced_to_single_undo: simulates 5-frame gizmo drag,
    verifies 1 undo entry, verifies undo restores original position
  - color_preset_click_creates_one_undo_entry: instant click → 1 undo entry
  - consecutive_drags_create_separate_undo_entries: 2 drags → 2 entries,
    verifies correct undo ordering with position verification

      Rotation displayed in degrees, applied in radians.
    - Material section (shown when all nodes have color): absolute-value editing for

- src/ui/dock.rs: Updated properties::draw() call to pass &selected_set from node_graph_state.

Design decisions:
- Delta-based transforms: position/rotation use drag values starting at zero. Each frame's
  non-zero delta is applied to all selected nodes. This is the industry-standard approach
  (Blender, Maya) — avoids overwriting different positions with one absolute value.
- Absolute material values: color/roughness/metallic/fresnel set the same value on all
  selected nodes. This matches industry convention — material is typically set uniformly.
- First-node display: material sliders show the first selected node's current value as the
  starting point. Changing the slider applies the new value to all selected nodes.

- src/shaders/rendering.wgsl: Added scene light loop in Full PBR shading path (after
  global key/fill lights, before environment reflection). For each scene light:
  - Point lights: direction from surface to light position, inverse-square falloff with
    range-based windowing (smooth window = (1 - (d/r)^2)^2 / d^2).
  - Spot lights: same as point + cone falloff using cos(half_angle) with smooth inner/outer
    transition (inner cone at 80% of outer angle).
  - Directional lights: constant direction (negated), no attenuation.
  - Full Cook-Torrance GGX BRDF (D_GGX, G_SmithGGX, F_Schlick) evaluated per scene light,
    same quality as global lights.

  collect_scene_lights(), packs into flat arrays, passes to render uniform. Pick uniforms
  pass zeroed light data (pick doesn't need lighting).
- src/app/async_tasks.rs: Screenshot uniform now includes scene light data.

Design decisions:
- Packed lights into camera uniform buffer instead of separate storage buffer. 8 lights ×
  4 vec4f = 512 bytes is well within uniform limits and avoids new bind group complexity.
- Distance-sorted culling: when >8 lights exist, the 8 nearest to camera are active.
  This is the industry-standard forward rendering approach.
- Range-based windowing: smooth quadratic falloff avoids hard light cutoff at range boundary.
- Spot inner cone at 80% of outer: provides natural smooth falloff without extra parameters.

  cone falloff. When a light has a cookie (scene_light_vol[li].z > 0.5), reads the cookie

Design decisions:
- CPU-side expression evaluation: expressions are evaluated once per light per frame on the CPU,
  not per-pixel on the GPU. This is correct because the expressions only affect per-light
  properties (intensity, color), not per-pixel values. Zero GPU cost.
- Parse on every frame: the expression string is re-parsed each frame. For the tiny expressions
  involved (typically <20 tokens), parsing takes <1μs. Caching the parsed AST would require
  invalidation logic for when the user edits the text field — not worth the complexity.
- hue_rotate_color reuse: the existing hue rotation function (from Array lights) is reused for
  color hue expressions. Degrees-based, matching the Rainbow preset (0-360 range).
- Expression errors shown inline: invalid expressions show a red error message in the properties
  panel. The static value is used as fallback when parsing fails — no visual disruption.
